## 一、妙手ERP 基本画像

### 1.1 公司背景

| 维度 | 信息 |
|------|------|
| 公司全称 | 深圳呈云网络科技有限公司 |
| 产品名称 | 妙手ERP (erp.91miaoshou.com) |
| 成立时间 | 2014年始创 |
| 团队规模 | 数百人技术研发团队，核心成员来自阿里、腾讯等 |
| 融资情况 | 未公开披露具体融资轮次（可能自筹或低调融资） |
| 注册用户 | 110万+（2025年数据，2024年为80万+） |
| 总部位置 | 深圳 |

### 1.2 核心产品定位

妙手ERP定位为**跨境电商一体化运营SaaS**，面向中小跨境卖家，覆盖"采集-翻译-刊登-定价-营销-订单-仓储-采购-物流-财务-客服"全链路。主打特色是**基础功能永久免费**，以低门槛获客。

### 1.3 支持平台（30+个）

**核心平台**：Shopee、Lazada、TikTok Shop、Temu、SHEIN、Ozon、Wildberries、Amazon、eBay、AliExpress、Wish、Walmart、Allegro、Coupang、Mercado Libre 等。

**采集货源**：1688、淘宝、拼多多等100+国内外货源平台。

**关键确认：妙手ERP 明确支持 1688采集 + Ozon上架 这条完整链路。**

### 1.4 免费版 vs 付费版

| 子账号 | 3个 | 更多 | 更多 |
| 订单数据存储 | 120天 | 更长 | 更长 |
| 产品刊登 | 不限数量（部分平台除外） | 不限 | 不限 |
| 关键词采集 | 不支持 | 支持 | 支持 |
| 价格检测 | 不支持 | 支持 | 支持 |
| 定时发布 | 不支持 | 支持 | 支持 |
| 产品诊断 | 不支持 | 支持 | 支持 |
| 图片空间 | 300MB | 更大 | 更大 |
| Ozon单店月费 | -- | 约20元/月 | 含在套餐内 |

### 1.5 用户规模与口碑

- 用户评分：4.7/5（基于2000+企业反馈，2025Q3数据）
- 主要优势：免费入门门槛低、操作简单、自动化程度高、客服响应快
- 主要不足：
  - 免费版限制较多（仅1个店铺、不支持关键词采集）
  - 多平台订单无法合并批量采购
  - 高级功能（智能定价等）需付费
  - 翻译质量为通用机翻级别，俄语电商专业度有限

## 二、妙手ERP 的 1688->Ozon 链路能力详析

### 2.1 1688采集能力

| 采集方式 | 支持情况 | 说明 |
|----------|---------|------|
| 链接采集 | 支持 | 复制1688产品链接到输入框即可采集 |
| 关键词采集 | 付费版支持 | 免费版不支持 |
| 整店采集 | 支持 | 可抓取整个1688店铺 |
| 浏览器插件采集 | 支持 | Chrome插件，一键采集到ERP |
| 1688跨境分销一键铺货 | 支持 | 妙手首家接入1688跨境分销轻应用 |
| 导入采集 | 支持 | 批量导入链接 |

### 2.2 采集数据字段

根据调研，妙手从1688采集的数据包含：
- 商品标题（中文）
- 商品价格（含阶梯价）
- 商品主图/详情图
- 商品属性/规格
- SKU变体信息（颜色/尺寸等）
- 供应商信息

**未确认是否采集**：供应商评分、发货时效、最小起订量、历史销量等深层数据。

### 2.3 翻译能力

| 能力 | 详情 |
|------|------|
| 自动翻译俄文 | 支持，内置AI翻译引擎 |
| 翻译引擎 | 多引擎可选：AI翻译、阿里翻译、象译等 |
| 批量翻译 | 支持，可一次性翻译多个Listing |
| 图片翻译 | 支持，自动识别图中文字翻译为俄语，0.035元/张 |
| 翻译质量评估 | 通用机翻级别，电商语料库优化过，但与人工翻译/专业AI有差距 |

**关键局限**：翻译质量是"能用"级别，而非"优秀"级别。对俄语电商SEO关键词、文化适配、Ozon标签系统的针对性优化能力有限。

### 2.4 Ozon品类匹配

- 支持批量设置类目（category mapping）
- 需要人工选择或半自动匹配
- **不具备智能品类推荐/自动映射能力**（需人工操作）

### 2.5 Ozon属性填充

- 支持批量编辑属性
- 基础属性可从1688数据映射
- **Ozon必填属性需要人工填写或半自动处理**
- 不具备自动理解1688属性并映射到Ozon属性体系的智能能力

### 2.6 上架能力

- 支持一键上架到Ozon
- 支持批量上架
- 批量处理能力：可同时处理大量商品（具体上限未公开，但支持数百级别）
- 上架前支持批量编辑、批量定价

### 2.7 选品推荐/评分功能

| 功能 | 妙手ERP |
|------|---------|
| AI选品 | 有，但偏向热销榜单推荐 |
| 多维度评分体系 | 无专业评分模型 |
| 竞品分析 | 无（需配合Seerfar/MPSTATS等外部工具） |
| 利润计算 | 有定价模板，自动计算售价，利润核算误差率<0.3% |
| 市场趋势分析 | 无深度分析，仅有热销数据 |
| 选品与Ozon市场数据联动 | 无 |

### 2.8 总结：妙手ERP 1688->Ozon 能做什么 / 不能做什么

**能做的（约60-70%的基础链路）**：
1. 从1688批量采集商品数据
2. 自动翻译标题/描述为俄语
3. 图片翻译（中文转俄文）
4. 批量设置定价模板
5. 批量上架到Ozon
6. 订单管理和发货处理
7. 基础利润核算

**做不到或做得不好的（30-40%的高价值环节）**：
1. 无智能选品评分模型（无法告诉你"该选什么"）
2. 无Ozon市场数据驱动的选品决策
3. 无1688供应商质量评估
4. 俄语翻译质量有限，非电商SEO级别
5. 无Ozon标签(Tag)系统的智能优化（2025年Ozon用标签替代关键词）
6. 无自动品类智能映射（需人工）
7. 无Ozon必填属性的智能推断和填充
8. 无竞品定价分析
9. 无差异化listing生成能力（千篇一律的机翻listing）

### 3.2 各ERP分析

**妙手ERP** -- 免费版入门门槛最低，1688采集方式最丰富，Ozon支持完整但不深。适合铺货型卖家快速起步。

**店小秘** -- 老牌ERP，平台覆盖广，Ozon模块完善。支持ChatGPT一键生成产品信息。需VIP才能用Ozon。面向中大卖家。

**马帮ERP** -- 强项在海外仓和物流管理，Ozon基础功能齐全。适合有海外仓业务的卖家。

**芒果店长** -- Ozon功能较深，支持多SKU刊登、批量上传、自动调价、售后回复等。操作简单，适合新手。是Ozon官方推荐的ERP合作伙伴。

**赛狐ERP** -- 专注亚马逊，不支持Ozon，与本项目无关。排除。

**积加ERP** -- 已对接Ozon/Wildberries/Yandex三大俄罗斯平台，支持商品管理、订单管理、库存同步。偏精品运营模式。

### 3.3 Ozon支持深度排名

1. **芒果店长** -- Ozon官方合作伙伴，功能最深
2. **积加ERP** -- 三大俄罗斯平台全覆盖
3. **妙手ERP** -- 基础功能完善，价格最低
4. **店小秘** -- 功能完善但需付费
5. **马帮ERP** -- 基础对接
6. **赛狐ERP** -- 不支持

## 四、差异化分析：自建系统的价值空间

### 4.1 妙手ERP做不到但我们能做的事

| 差异化能力 | 说明 | 价值等级 |
|-----------|------|---------|
| **AI智能选品评分** | 多维度评分模型（市场需求+竞争度+利润率+供应商质量+物流可行性），输出"该选什么" | 极高 |
| **Ozon市场数据驱动决策** | 对接Seerfar/MPSTATS等数据源，用Ozon实际销售数据指导选品 | 极高 |
| **1688供应商深度评估** | 供应商信誉评分、发货时效、退款率、历史合作数据 | 高 |
| **AI原生俄语listing生成** | 非翻译，而是基于产品理解直接生成俄语电商文案，针对Ozon SEO和标签系统优化 | 极高 |
| **Ozon品类智能映射** | 自动将1688商品属性映射到Ozon品类树和必填属性 | 高 |
| **竞品定价策略** | 分析Ozon同品类竞品价格带，给出最优定价建议 | 高 |
| **知识库沉淀** | 产品知识库持续积累，形成选品经验数据资产 | 中 |

### 4.2 妙手ERP做得不好但我们能做更好的事

| 环节 | 妙手ERP现状 | 我们能做到 |
|------|-----------|-----------|
| **俄语翻译** | 通用机翻，千篇一律 | AI原生生成，针对Ozon标签系统优化，融入俄语消费者搜索习惯 |
| **品类映射** | 手动选择品类 | 自动识别商品类型，智能匹配Ozon品类，自动填充属性 |
| **定价策略** | 成本+利润率公式 | 结合Ozon竞品价格带+物流成本+平台费用的智能定价 |
| **选品逻辑** | 热销榜单推荐（跟卖思维） | 数据驱动的蓝海选品（差异化思维） |
| **Listing质量** | 翻译即上架 | 多语种SEO优化+卖点提炼+文化适配 |

### 4.3 "站在妙手ERP肩膀上"的可能性

**结论：这是最优策略。**

理由：
1. **妙手ERP的强项恰好是我们不应该自建的模块** -- 订单管理、物流对接、仓储管理、批量操作UI等"重运营"模块，妙手做了10年，我们没必要重做
2. **我们的强项恰好是妙手ERP的短板** -- 智能选品、AI文案生成、数据驱动决策等"重智能"模块，是ERP工具的天然盲区
3. **成本效益**：自建一个完整ERP需要数百人年的开发量，而用妙手ERP的基础功能（免费或低成本）+ 自建智能层，是性价比最高的方案

## 五、最终产品策略建议

### 5.1 核心结论

**不要全部自建，也不要全部用ERP。采用"智能层自建 + 运营层用ERP"的混合架构。**

### 5.2 具体建议

#### 自建模块（我们的核心竞争力）

| 模块 | 优先级 | 当前状态 | 说明 |
|------|--------|---------|------|
| AI选品评分引擎 | P0 | 已有ozon-evaluator.js | 继续强化多维度评分模型，融入Ozon市场数据 |
| 1688数据深度抓取 | P0 | 已有source-1688-lib.js | 保留，抓取ERP拿不到的深层数据（供应商评分、历史销量等） |
| AI俄语Listing生成 | P0 | 已有ozon-listing-draft-flow.js | 核心差异化，直接生成而非翻译 |
| Ozon品类/属性智能映射 | P1 | 已有listing-attributes.json | 自动匹配品类树+填充必填属性 |
| 产品知识库 | P1 | 已有knowledge-base/ | 持续积累，形成数据壁垒 |
| 竞品价格分析 | P1 | 待建 | 对接Seerfar/MPSTATS的Ozon数据 |
| 供应商质量评估 | P2 | 待建 | 1688供应商多维评分 |

#### 交给ERP做的模块（用妙手或芒果店长）

| 模块 | 推荐ERP | 说明 |
|------|---------|------|
| 批量上架到Ozon | 妙手ERP / 芒果店长 | 成熟功能，无需自建 |
| 订单管理 | 妙手ERP | 多店铺订单统一管理 |
| 物流发货 | 妙手ERP | 对接490+物流商 |
| 仓储管理 | 妙手ERP | 对接800+仓储商 |
| 基础翻译（作为兜底） | 妙手ERP | 我们的AI生成为主，ERP翻译为辅 |
| 财务核算 | 妙手ERP | 利润核算误差率<0.3% |
| 客服工具 | 妙手ERP | 基础售后处理 |

### 5.3 核心竞争力定位

我们的系统定位不是"另一个ERP"，而是**"跨境选品智能决策引擎"**。

核心价值主张：
> "告诉你该卖什么、怎么定价、怎么写listing，让ERP去执行上架和运营。"

与ERP的关系：
- 不是竞争关系，是**上下游互补关系**
- 我们是"大脑"（决策层），ERP是"手脚"（执行层）
- 未来可以做成SaaS，面向所有使用妙手/芒果店长的Ozon卖家

### 5.4 短期执行计划（1-3个月）

1. **立即**：注册妙手ERP免费版，实测1688->Ozon全链路，记录痛点
2. **第1个月**：优化AI选品评分模型，集成Ozon市场数据（Seerfar API/爬取）
3. **第2个月**：强化AI俄语Listing生成质量，针对Ozon 2025新标签系统优化
4. **第3个月**：完成品类智能映射模块，实现"选品->生成listing->导出到ERP上架"闭环

### 5.5 长期愿景（6-12个月）

- 将系统产品化为SaaS，面向Ozon卖家群体
- 与妙手ERP/芒果店长做API对接，实现无缝数据流转
- 建立选品知识图谱，形成数据壁垒
- 扩展到Wildberries、Yandex Market等俄罗斯平台

## 六、风险与注意事项

1. **妙手ERP的API是否真正开放**：虽然声称200+接口，但需实际验证是否支持我们需要的数据回写场景
2. **免费版限制**：1个店铺的限制可能不够，建议预算付费版（Ozon单店20元/月，成本可控）
3. **ERP厂商也在进化**：妙手/芒果店长未来可能增加AI选品功能，但短期内（12个月内）不太可能做到我们的深度
4. **数据合规**：1688数据抓取需注意合规风险，优先使用1688官方开放的跨境分销接口
5. **Ozon平台变化**：2025年Ozon搜索从关键词转向标签系统，这是我们的机会窗口（ERP尚未适配）
